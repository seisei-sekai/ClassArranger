### 3. 工作流程依赖关系

工作流程中的步骤存在严格的依赖关系：
```
学生课时确认 → 讲师时间确认 → 教室确定 → 课表制作 
    → 讲师时间再次确认 → 学生时间确认 → 钉钉登录大屏幕 
    → 课表发放（讲师+学生） → 最终确认 → 学管确定
```

**规则**:
1. 前一步未完成，后续步骤不应开始
2. 所有布尔字段默认为False
3. 教室确定后才能制作课表
4. 课表发放需要同时发送给讲师和学生

### 8. 时间格式说明

系统中存在多种时间格式：

#### 录入日期
- 格式: 标准日期时间（datetime）
- 示例: `2025-12-05 00:00:00`

#### 起止时间（自由格式）
- 截止式: `~1210之前上完`、`12月24日之前上完`
- 日期式: `2025-12-06 00:00:00`
- 时间段式: `1206-1209`
- 具体时间: `12/13日周六14~16点`
- 区间式: `12月14日之后\n1月8号之前`

#### 希望具体时间（时间段描述）
- 时间点: `14点后`、`14:30之后`
- 时间段: `18—20`、`10～12`
- 限制描述: `平日需12：30之前，18点之后`
- 禁止时间: `12/25~1/5 不上课`

### 9. 课时系统说明

#### 三种课时来源
1. **已有现地课时**: 学生在日本当地已购买的课时
2. **国内ERP**: 学生在国内通过ERP系统购买的课时
3. **现地+ERP**: 混合使用两种课时

#### 课时编码格式
- **标准编码**: 8位数字，格式为`YYYYNNNN`
  - 示例: `20252413`
  - 解释: 2025年第2413个课时订单
- **ERP编码**: 格式为`ERP` + 数字
  - 示例: `ERP1023`
  - 解释: ERP系统中的第1023个订单
