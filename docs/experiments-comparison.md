## 技术栈对比

### 前端技术

```
Experiment:
- React Hooks (useState, useEffect)
- 自定义日历组件
- 简单CSS

Experiment2:
- React Hooks (10个useState)
- 自定义增强日历
- 日系风格CSS
- LocalStorage API

Experiment3:
- React Hooks + Context
- FullCalendar库
- 复杂组件系统
- Web Workers
- LocalStorage + IndexedDB (计划)
```

### 算法实现

```
Experiment:
- 贪心算法（简单实现）
- 遗传算法（基础版本）
- 时间复杂度: O(n²)

Experiment2:
- 贪心算法（优化版）
- 三方匹配（简化版）
- 时间复杂度: O(n × m × k)

Experiment3:
- 贪心算法（完整优化）
- 遗传算法（多代进化）
- 三方匹配（完整引擎）
- 时间复杂度: O(n × m × k) + 缓存
```

### 数据处理

```
Experiment:
- 内存存储
- 简单验证
- 无持久化

Experiment2:
- LocalStorage
- 中等验证
- 自动保存

Experiment3:
- LocalStorage + 缓存
- 完整验证
```

## 开发建议

### 选择标准

**选Experiment如果**:
- 你在学习排课系统
- 需要理解算法原理
- 想快速实验想法
- 数据量很小（<20）

**选Experiment2如果**:
- 你需要实际使用
- 用户是非技术人员
- 需要真实业务数据支持
- 希望快速上手

**选Experiment3如果**:
- 你需要完整功能
- 业务场景复杂
- 需要AI辅助
- 数据量大（>50）

### 组合使用

```
开发流程:
1. Experiment - 学习和原型
2. Experiment2 - 日常使用
3. Experiment3 - 高级场景

测试流程:
1. Experiment - 算法测试
2. Experiment2 - 功能测试
3. Experiment3 - 压力测试
```
